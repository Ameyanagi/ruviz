# ruviz v0.4.2

Released: 2026-04-08

## Highlights

- The Python API now supports live `ObservableSeries` arithmetic and NumPy ufunc derivations, so notebook workflows can express reactive transforms directly from mutable data sources.
- `copy(plot)` and `deepcopy(plot)` now preserve independent native plot state, while copied observables keep their own live listeners and values instead of aliasing shared state.
- Python packaging now exposes explicit dataframe extras for pandas and Polars, and the docs/examples cover dataframe-backed plotting, reusable plot templates, and observable math.
- The notebook widget bundle is now rebuilt from the shared web SDK frontend, with `plot.size_px(width, height)` controlling the rendered widget aspect ratio.

## Version Alignment

This release keeps the published workspace packages aligned on one version:

- `ruviz` `0.4.2`
- `ruviz-web` `0.4.2`
- `ruviz-gpui` `0.4.2`
- npm `ruviz` `0.4.2`
- PyPI `ruviz` `0.4.2`

## Python And Notebook Notes

- `ObservableSeries` now supports elementwise scalar/observable arithmetic plus NumPy ufunc composition, and derived observables stay live until they are written to directly.
- The public Python docs now document `pip install "ruviz[dataframes]"`, `deepcopy(plot)` template workflows, dataframe column selection, and the new observable math patterns.
- The package source root now lives under `python/ruviz_py`, keeping the bundled anywidget frontend and Python package layout aligned with `maturin`.
- The notebook widget frontend is generated from `packages/ruviz-web/src/python-widget.ts`, consolidating the widget build with the shared web runtime source tree.

## Packaging And CI

- Updated the workspace package versions, package READMEs, and guide snippets to `0.4.2`.
- Python CI now invokes `python -m pytest` and `python -m ruff check` explicitly, matching the packaged tool entrypoints used in contributor environments.
- Generated-preview CI is now scoped to preview-relevant path changes and split across Rust, Python, and web lanes to reduce unnecessary release-adjacent rebuild work.

## Quality And Validation
