# ruviz v0.1.5

Released: 2026-03-23

## Highlights

- Added the new `ruviz-gpui` workspace crate for GPUI-based component integrations.
- Added GPUI interactive session support and reactive plotting flows for embedded and streaming integrations.
- Improved interactive and reactive rendering internals used by the GPUI adapter.
- Hardened DPI handling, reactive snapshot validation, export overwrite behavior, and non-Cartesian bounds calculation.
- Changed the default tick-mark layout to render on all four sides of the plot frame.

## GPUI Integration

This release prepares the workspace to publish both crates from the repository:

- `ruviz`
- `ruviz-gpui`

The release workflow now publishes `ruviz` first, waits for crates.io to serve the new version, and then publishes `ruviz-gpui`.

Notable GPUI-path fixes in this release:

- Correct interactive invalidation and streaming redraw behavior.
- Refresh overlays and reactive source setters consistently.
- Preserve manual axis limits and tighter DataShader bounds in reactive flows.

## Compatibility Notes

- Default tick marks now render on all four sides of the plot frame.
- To preserve the previous bottom-left-only appearance, call `.ticks_bottom_left()`.
