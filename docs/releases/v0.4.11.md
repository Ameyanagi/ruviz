# ruviz v0.4.11

Released: 2026-04-12

## Highlights

- The published npm package now includes the wasm runtime under `generated/raw`, fixing `bun install ruviz` and `npm install ruviz` for browser and wasm consumers.
- Release validation now checks the real `npm pack` tarball and smoke-installs it before publish, so future npm releases cannot silently omit the wasm payload.
- Browser-side repeated unchanged exports and identical `setPlot(plot)` attaches now reuse cached wasm-side state instead of rebuilding the same plot repeatedly.

## Version Alignment

This release keeps the published workspace packages aligned on one version:

- `ruviz` `0.4.11`
- `ruviz-web` `0.4.11`
- `ruviz-gpui` `0.4.11`
- npm `ruviz` `0.4.11`
- PyPI `ruviz` `0.4.11`

## npm Packaging Recovery

- Added a package-local npm ignore configuration so npm packaging no longer inherits the repo-root ignore rules that exclude generated wasm assets.
- The wasm build scripts now emit `generated/raw/.npmignore`, preventing wasm-pack and wasm-bindgen generated `.gitignore` files from stripping the runtime from the tarball.
- Added `packages/ruviz-web/scripts/verify-npm-package.mjs` to verify the real tarball contents and smoke-install that tarball with `bun add`.

## Interactive And Wasm Tooling

- Added a dedicated Rust/wasm interactive benchmark suite with recorded smoke baselines for hover, pan, time, `setPlot`, and export flows.
- Static `PlotBuilder` exports and unchanged browser session exports now reuse cached wasm-side state, and identical `setPlot(plot)` calls are treated as no-ops.

## Quality And Validation

This release was locally validated after the `0.4.11` version bump with:

- `cargo check -q`
- `bun run check:web`
- `bun run --cwd packages/ruviz-web verify:pack`
- `cd python && uv run cargo test`
- `cd python && uv run python -m pytest`
