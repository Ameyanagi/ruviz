# ruviz v0.3.0

Released: 2026-04-01

## Highlights

- Added Linux and Windows desktop support for `ruviz-gpui`, so the embedded GPUI adapter now has a first-class native path across all three major desktop platforms.
- Raised the repository MSRV to Rust `1.92` and kept a dedicated CI lane to enforce that floor while contributors default to the latest stable toolchain locally.
- Strengthened release confidence with explicit `ruviz-gpui` example compilation checks and shared Linux desktop dependency installation across CI and release jobs.
- Moved the temporary GPUI patching flow onto an upstream `zed-industries/zed` revision and kept the workspace aligned on the same fix set.

## Version Alignment

This release keeps the published workspace packages aligned on one version:

- `ruviz` `0.3.0`
- `ruviz-web` `0.3.0`
- `ruviz-gpui` `0.3.0`
- npm `ruviz` `0.3.0`

## Desktop And Toolchain Notes

- `ruviz-gpui` now supports Linux, macOS, and Windows.
- Linux desktop builds still require GTK3 development headers for native file dialogs.
- GPUI examples now emit clearer hints when launched from headless Linux or Windows sessions.
- The repository MSRV is Rust `1.92`; `rust-toolchain.toml` tracks the latest stable compiler for day-to-day contributor workflows, and CI enforces the minimum supported floor separately.

## Internal And Release Engineering

- Refactored oversized plotting, rendering, and observable modules into smaller internal submodules while preserving public module paths.
- Deduplicated Linux desktop package installation for CI and release workflows through a shared composite action.
- Kept release packaging verification aligned across crates.io, npm, docs, and multi-platform CI.
