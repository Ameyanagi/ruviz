# ruviz v0.5.0

Released: 2026-07-17

## Highlights

- Streaming data can now be replaced atomically with `StreamingXY::replace`, and acknowledgement watermarks are tracked per consumer so multiple sessions or exports sharing one stream never composite stale frames.
- Outside legend positions are honored end to end: layout reserves a side band for `OutsideRight`, `OutsideLeft`, `OutsideUpper`, and `OutsideLower`, accounting for labels, DPI, margins, and colorbars across PNG, SVG, parallel, interactive, and subplot rendering.
- Interactive annotations can be added, updated, and removed at runtime through session-scoped `AnnotationId`s, with overlay-only invalidation that reuses cached base geometry.
- The GPUI adapter gains coordinate mapping, pointer events, and `set_plot_keep_view` for replacing plot content while preserving a customized pan/zoom view.

## Streaming And Interaction

- `StreamingXY::replace` swaps a stream's entire contents in one call: input is collected before locks, truncated to capacity, and committed with a single notification and a full-redraw acknowledgement. Empty streams remain renderable, including on log axes and through viewport snapshots taken before the first render.
- Streaming acknowledgement watermarks are per consumer: each interactive frame cache records the exact stream sequence its base image contains, so one consumer acknowledging a replacement no longer lets another consumer take the incremental path over stale data.
- Movable interactive annotations support every `Annotation` variant with fallible add/query/update/remove, coordinate validation that rejects non-positive values on logarithmic axes, and dirty tracking limited to the overlay.
- Heatmaps accept a configurable row origin, `HeatmapOrigin::{Upper, Lower}`, applied consistently to rendering, cell bounds, and interactive hit testing, including reversed Y axes.

## API Additions

- The `BuilderWhen` trait adds a conditional `.when(condition, |builder| ...)` combinator to plot, series, group, subplot, config, theme, and interactive window builders. Implementations are per builder family, so `use ruviz::prelude::*` coexists with GPUI's `FluentBuilder::when`.
- `RuvizPlot::set_plot_keep_view` replaces plot content while preserving customized visible bounds; ordinary `set_plot` keeps its documented destructive reset semantics.
- Subplot suptitles are measured during layout, with a title-size API and clarified spacing semantics.

## Compatibility

- `PlotLayout` and `MeasuredDimensions` regained external struct-literal constructibility: legend placement state introduced during outside-legend work moved to crate-internal composition types, restoring the exact v0.4.20 public field sets. An external-crate regression test guards the contract.
- This release is a minor version bump because `cargo semver-checks` against the published `ruviz 0.4.20` baseline reports two API-breaking additions:
  - `HeatmapConfig` gained the public `origin` field, so exhaustive struct literals of `HeatmapConfig` must add it (builder-style construction via `HeatmapConfig::new()` is unaffected).
  - `PlottingError` gained the `InvalidAnnotation` and `UnknownAnnotationId` variants, so exhaustive matches on `PlottingError` must handle them (matches with a `_` arm are unaffected).

## Documentation And Quality Gates

- The Rust gallery is kept fresh deterministically: `generate_gallery --check` validates generated Markdown, copy-backed byte identity, catalog integrity, and links without writing; `make rust-gallery` and `make check-rust-gallery` wrap generation and read-only validation; a path-scoped CI job enforces both.
- Subplot gallery figures were renamed and clarified, scientific showcase gutters tightened, the international figure gained an English reference panel, and gallery source labels became repository-relative links.
