# ruviz v0.4.4

Released: 2026-04-08

## Highlights

- PyPI now publishes a manylinux `x86_64` wheel for `ruviz`, so common Linux `uv add ruviz` installs can use a prebuilt wheel instead of falling back to a source build.
- Python source distributions now include the workspace `benches/` tree required by the root Cargo manifest, fixing the source-build failure that blocked installs when no compatible wheel was available.
- The GitHub release workflow now publishes from tag refs only, including manual recovery runs pinned to `refs/tags/<tag>`.

## Version Alignment

This release keeps the published workspace packages aligned on one version:

- `ruviz` `0.4.4`
- `ruviz-web` `0.4.4`
- `ruviz-gpui` `0.4.4`
- npm `ruviz` `0.4.4`
- PyPI `ruviz` `0.4.4`

## Python Packaging Recovery

- Release CI now builds and publishes a Linux wheel alongside the existing macOS, Windows, and sdist artifacts.
- CI now smoke-tests both the generated sdist install path and the Linux wheel artifact before a release can publish.
- The published Linux wheel keeps static rendering and notebook widgets available; install from source on Linux if you need the native desktop `plot.show()` window.

## Release Workflow

- `.github/workflows/release.yml` now serializes release runs per tag and always checks out the tagged ref during manual recovery dispatches.
- `docs/releases/README.md` now documents the supported release path: merge release prep, push the tag, and let GitHub Actions publish the crates, npm package, Python distributions, and GitHub Release.

## Quality And Validation

This release was locally rechecked after the `0.4.4` version bump with:

- `cargo check -q`
- `bun run check:web`
- `cd python && uv run cargo test`
- `cd python && uv run python -m pytest`

Targeted packaging smoke checks also rebuilt and installed the generated sdist plus a reduced-feature wheel locally.
