# ruviz v0.3.2

Released: 2026-04-01

## Highlights

- Completed the unified tag-driven release flow across crates.io, npm, GitHub Releases, and PyPI after fixing the Python packaging issues uncovered during the `0.3.1` release.
- Updated the Python release builders to use `maturin` `1.12.6`, which fixes the duplicate `README.md` conflict that broke source distribution generation with `maturin` `1.11.5`.
- Switched the macOS Intel wheel lane to the supported `macos-15-intel` GitHub-hosted runner so PyPI publishes both Intel and Apple Silicon macOS wheels again.

## Version Alignment

This release keeps the published workspace packages aligned on one version:

- `ruviz` `0.3.2`
- `ruviz-web` `0.3.2`
- `ruviz-gpui` `0.3.2`
- npm `ruviz` `0.3.2`
- PyPI `ruviz` `0.3.2`

## Python Packaging Notes

- PyPI publishing runs from `.github/workflows/release.yml` using GitHub OIDC trusted publishing.
- Release tags remain `vX.Y.Z`, while Python prereleases map to PEP 440 versions such as `X.Y.Zrc1`.
- Linux wheels intentionally use host-native `linux` compatibility because the current Python package keeps desktop interactive support enabled.
